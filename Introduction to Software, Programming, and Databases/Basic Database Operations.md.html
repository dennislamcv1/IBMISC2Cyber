Basic Database Operations

Estimated Time Needed: 45 min

In this lab, you will create a database, import data to populate a table, and perform basic database operations in the MySQL database service using the phpMyAdmin graphical user interface (GUI) tool.


Objectives

After completing this lab, you will be able to use phpMyAdmin with MySQL to:

  • Create and populate a database and table
  • Execute Structured Query Language (SQL) commands to perform basic database operations
  • Retrieve data from tables using simple SELECT statements
  • Learn how to filter the data output using WHERE statements
  • Add and delete data from a table using ADD and DELETE statements

Software used in this lab


You will use MySQL to complete this lab. MySQL is a free, open-source relational database system that offers a command line interface (MySQL) and a third-party web interface (phpMyAdmin) to efficiently store, manipulate, and retrieve data.

MySQL is a service available on Skills Network Labs (SN Labs) Cloud IDE, a virtual lab environment used in this course. SN Labs Cloud IDE is great way to do projects without downloading, installing, configuring, and integrating software on your own computer.


Two Components of the SN Labs Cloud IDE

  2. The area on the right side of the screen is where you will use the menus, terminals, and tools to complete the lab exercises.

Exercise 3: Select data from a database

Structured Query Language (SQL) Statements, also known as "commands", are used to communicate with a database to retrieve data, or modify its structure and its contents.

The SELECT statement is a database query that is used to request data from the relevant rows and columns in a table. The output of the query is stored and displayed in a table known as the SQL result-set.

Syntax of a SELECT Statement

SELECT column1, column2
FROM table_name;
  • The first line selects the field/column names of the data to be displayed.
  • The second line indicates the name of the table where the selected data is stored.

Task A: Review the Source Code

Step 1: In the phpMyAdmin screen, select the "SQL" tab from the top menu to access the query box.

Step 2: Copy the code block below and paste it into the query box.

SELECT *
FROM Policy;

  • The asterisk ( * ) in the first line of this SELECT statement is a wildcard character used to indicate that data from all columns should be returned in the query output.

Step 4: Review the query output. When these steps are completed correctly, the query output will display all 6 columns and 10 rows of data from the Policy table.

Exercise 4: Filter data output

The WHERE statement defines a condition used to filter records and limit the amount of data returned in the query output.

Sytnax of a WHERE Statement

SELECT column1, column2, ...
FROM table_name
WHERE condition;

  • The first line selects the field/column names of the data to be displayed.
  • The second line indicates the name of the table where the selected data is stored.
  • The third line defines the conditon to test and limit the data in the query output.

Task A: Review the Source Code

Step 1: Select the "SQL" tab from the top menu to access the query box.

Step 2: Use the "Clear" button to delete the previous SQL Statement.

Step 3: Copy and paste the code block below to the query box to filter and display the records from the Policy table that have Masonry construction.

SELECT *
FROM Policy
WHERE ConstructionType = "Masonry";

Step 4: Select the "Go" button.

Step 5: Review the query output.

  • When these steps are completed correctly, the query output will display 6 columns and 6 rows of data from the Policy table.

Exercise 5: Add a new row to a table

To add a new record/row to a table, use the SQL statement INSERT INTO.

Syntax of an INSERT INTO Statement

INSERT INTO table_name
VALUES (value1, value2, value3, ...);

  • The first line indicates the name of table where the new record will be added.
  • The second line assigns values in the same order as the columns in the table.

Task A: Review the Source Code

Step 1: Select the "SQL" tab from the top menu to access the query box.

Step 2: Use the "Clear" button to delete the previous SQL Statement.

INSERT INTO Policy

Exercise 6: Delete a row from a table

The DELETE statement is used to delete existing records from a table.

Sytnax of a DELETE Statement

DELETE FROM table_name 
WHERE condition;

  • The first line indicates the name of the table that records will be deleted from.
  • The second line defines the conditon and deletes the record if the condition is true.
